grid(ism)

An experiment in responsive CSS grids.

grid(ism) aims to create a responsive CSS grid for creating website layouts. It aims to be robust, practical, and easy to get your head around.

Priciples

Here’s what I want out of a grid system.

  1. The system should allow for fluidity, while remaining predictable across devices and resolutions.
  2. It should not be necessary to explicitly declare rows, or label which horizontal sections are the first in the row.
  3. Each horizontal section should be able to be partitioned and controlled independantly from other sections.
  4. Margins on the outside should be the same as internal gutters at all times.
  5. The names of classes and usage instructions should be simple, and memorable.

Usage

Include the following css in your page:

<link rel="stylesheet" href="css/gridism.css">
<link rel="stylesheet" href="css/gridism-responsive.css">

The responsive stylesheet is completely optional.

The Grid(ism) class

  • Anything with a class gridism will be a centered, horizontal section of the page, within which any of the grid classes can be laid out.
  • You may wish to use multiple gridism classes throughout your page, with each one inside it’s own full-width section of the page. Use your web-inspector on this page to see how this is done.

Horizontal Sections

  • Elements with a class of whole will be full-width.
  • Elements with classes of half or quarter will work together within a row.
  • Elements with classes of quarter or three-quarters will work together within a row.
  • Elements with classes of third or two-thirds will work together within a row.
  • Elements with classes of golden-small or golden-large will work together within a row. The widths of these classes correspond to the Golden Ratio.
  • Elements with classes of fifth or two-fifths, three-fifths, or four-fifths will work together within a row.
  • It is up to you to make sure each row is filled with enough horizontal sections to make up a complete row. Failure to do so will result in unexpected layouts.

Gutters, and wrapping

  • Gutters are created by using elements with a content class within any horizontal section.
  • By default, the gridism classes are full-width. If you’d like there to be some wrapping at a nice maximum width value on larger screens, simply place the gridism container inside an element with a wrap class. For example, you could add this as a class on the body tag or on any other element.

Responsiveness

  • On smaller devices, the horizontal sections will stop being placed side-by-side with neighboring classes and instead will be stacked vertically at full-width instead.
  • When the wrap class is used, the max-width value for the centered section on larger screens will adjust to make good use of the available screen space.
  • Below a certain threshhold, the wrap will transition to a full-width, fluid layout.
